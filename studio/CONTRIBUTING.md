# Contributing

Contributions, ideas, fixes, and hardware testing feedback are welcome.

Before submitting major architectural or workflow changes, please open an issue or discussion first so the direction can be reviewed against the current Spine and project goals.

## Development Rules

- Keep Studio, Firmware, and Tooling ownership separated.
- Do not embed React/web runtimes into firmware.
- Preserve upstream OpenChakra attribution and license headers.
- Avoid uncontrolled large-scale editor rewrites.
- Keep ESP32-P4 and LVGL workflows as the primary project direction.

## Pull Requests

Please:
- keep pull requests focused and readable
- document major behavior changes
- update README or Spine documentation where relevant
- avoid committing build artifacts or temporary files

## Community Expectations

Be respectful and constructive.

Harassment, abuse, discrimination, or intentionally disruptive behavior will not be tolerated.

This project is intended to be an open and welcoming environment for embedded developers, makers, and learners.
