KNOWN LIMITATIONS

ForgeUI RC1 is focused on stable hardware bring-up,
clean architecture, and reusable subsystem patterns.

RC1 intentionally prioritises:

- runtime stability
- hardware proof
- backend ownership separation
- safe LVGL patterns
- modular subsystem structure

Current known limitations include:

- runtime theme switching not implemented yet
- full SD formatting intentionally disabled
- admin authentication still early-stage
- no persistent WiFi credential storage yet
- no NTP sync yet
- no cloud sync implementation yet
- no SPIFFS database layer yet
- limited animation/polish focus in RC1
- no production OTA/update system yet

Current focus remains:

- stable ESP32-P4 baseline
- reusable architecture
- hardware-proven runtime systems
- clean framework ownership boundaries
